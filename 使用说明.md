# 使用说明

## 工作区

系统提供四个工作区，通过左侧导航栏切换：

| 工作区 | 用途 |
|--------|------|
| 对话 | 与 AI 助手对话，调用工具完成运维任务 |
| 任务 | 运行即时脚本或托管长期监控任务 |
| 设备 | 管理设备资产，查看存活状态 |
| 总览 | 查看运行态势、告警与恢复事件 |

---

## 对话

在对话框输入问题或指令，AI 助手会自动识别意图并调用对应工具。

- 支持多轮对话，上下文自动关联
- 点击左上角 **+** 新建对话
- 系统公告对话置顶显示，未读消息有红点提示

---

## 任务

### 即时任务

一次性执行，运行完即结束。适用于巡检、日志分析、诊断等场景。

### 托管任务

长期后台运行，持续输出状态。适用于监控、守护、轮询等场景。

托管任务状态说明：

| 状态 | 含义 |
|------|------|
| running | 正常运行中 |
| warning | 检测到异常 |
| recovered | 异常已恢复 |
| stopped | 已停止 |

---

## 设备

### 设备卡片

主页面以卡片形式展示所有设备，每张卡片显示设备名称、IP、类型和当前状态。

状态颜色：

| 颜色 | 状态 | 含义 |
|------|------|------|
| 绿色 | healthy | 设备在线，检测正常 |
| 黄色 | attention | 检测异常，连续失败 1~2 次 |
| 红色 | critical | 设备离线，连续失败 ≥ 3 次 |
| 灰色 | unknown | 尚未检测 |

### 新增设备

1. 点击右上角 **新增设备** 按钮
2. 填写设备名称、IP 地址、设备类型等信息
3. 保存后系统自动开始存活检测

### 编辑/删除设备

点击设备卡片进入详情，可编辑信息或删除设备。

### 筛选与搜索

- 顶部标签栏按设备类型筛选：全部 / 服务器 / 存储设备 / 安全设备 / 网络设备
- 搜索框支持按名称或 IP 模糊搜索

### 存活检测机制

- 检测方式：ping + tcp 端口连通性
- 检测间隔：5 秒
- 判定规则：连续失败 3 次标记为离线（critical），任一次成功即恢复

---

## 总览

### 运行态势

聚合展示所有托管任务的当前状态，快速定位异常任务。

### 最近事件

按时间线展示告警、恢复、启动、停止等事件。
