# AIops智能运维中枢 正式交付说明

## 1. 交付说明

本文档用于说明 AIops智能运维中枢 当前版本的交付范围、部署方式、能力边界和交付建议。  
适用于：

- 企业内部试点交付
- 部门级平台上线
- 运维/平台团队内部推广

---

## 2. 产品名称

**AIops智能运维中枢**

产品定位：

**面向企业内部运维场景的桌面端智能运维工作台**

核心理念：

- 对话即需求
- 脚本即功能
- 输出及交付

---

## 3. 当前交付版本定位

当前版本可定位为：

**企业内部一期可交付版本**

当前版本不是单纯的聊天应用，也不是演示型原型，而是一套具备以下核心能力的本地运维平台：

1. 自然语言驱动的运维工作入口
2. 基于 Skill 的本地能力映射
3. 即时任务执行
4. 托管任务值守
5. 系统通告与通知
6. MCP 外部能力接入
7. 审计记录、查询与回放

---

## 4. 当前交付范围

### 4.1 对话能力

支持：

- 普通大模型对话
- 多轮上下文
- 对技能能力的问答
- 对托管任务状态的查询

支持的典型问法包括：

- 你有什么 skill，能干什么
- 7001 端口最近怎么样
- service_watchdog 最后一次恢复是什么时候

### 4.2 Skills 能力

支持：

- 扫描项目内置 Skill
- 扫描用户目录 `~/.aiops/skills/`
- 上传 Skill
- 删除 Skill
- 编辑 Skill 说明和标签

### 4.3 即时任务

支持：

- 通过对话识别即时任务需求
- 执行本地即时 Python 脚本
- 返回结果到对话界面

### 4.4 托管任务

支持：

- 启动托管任务
- 停止托管任务
- 重启托管任务
- 配置托管任务参数
- 状态快照持久化
- 后台运行
- 应用重启后恢复
- 异常后受控自动恢复

### 4.5 系统通告

支持：

- 托管任务异常通知
- 托管任务恢复通知
- 独立系统通知会话
- 未读计数
- 清空通知

### 4.6 MCP

支持：

- 新增/删除 MCP Server
- 连接/断开 MCP Server
- 工具发现
- 工具执行
- 风险级别配置
- 工具风险覆盖配置

### 4.7 审计

支持：

- 路由决策审计
- 即时脚本执行审计
- 托管任务审计
- MCP 调用审计
- 查询
- 过滤
- 搜索
- 回放

建议交付前至少确认：

1. `cargo check` 通过

3. `npm run tauri build` 可产出 `.app`

5. Python 环境已验证
6. 至少一个即时任务和一个托管任务已实测
7. 系统通告链路已实测
8. 审计页已实测

## 8. 风险与边界说明

当前版本已具备交付条件，但仍存在明确边界。  
建议交付时同时提供风险说明。

核心边界包括：

1. 托管任务已具备守护能力，但还不是最终系统服务化形态
2. 审计中心已可用，但还不是完整合规审计平台
3. MCP 已做风险分级，但仍建议在受控环境中使用
4. 当前更适合单用户、本地桌面场景

完整风险说明请参考：

[AIops-风险与已知限制清单.md](/Users/meteor/code/AIops/docs/AIops-风险与已知限制清单.md:1)

## 9. 建议交付对象

当前版本建议优先交付给：

1. 企业内部运维团队
2. 平台工程团队
3. 研发支持团队
