# AIops智能运维中枢 功能回归检查清单

## 1. 说明

本文档用于 AIops智能运维中枢 在交付前进行功能回归检查。  
目标是确认当前一期版本的核心链路可用、稳定、可演示。

建议每次交付前至少完成以下三类检查：

1. 基础启动与配置检查
2. 业务功能回归检查
3. 安全与治理能力检查

---

## 2. 基础启动与配置检查

### 2.1 应用启动

- [ ] 应用可正常启动，无白屏、崩溃、明显卡死
- [ ] 左侧可正常看到品牌区、对话/脚本切换区、系统通告入口
- [ ] 默认首页不进入系统通告，而是进入聊天主页或最近普通会话
- [ ] 主题、背景色、侧栏折叠状态可正常恢复

### 2.2 模型配置

- [ ] 设置页可正常新增模型配置
- [ ] API Key 保存后可正常加载
- [ ] 模型列表可正常获取
- [ ] 模型切换后可正常发起对话
- [ ] 配置删除后不会残留异常状态

### 2.3 Python 环境

- [ ] 可正确识别 Python 解释器
- [ ] Python 路径修改后可生效
- [ ] 即时任务和托管任务均可调用 Python 脚本

### 2.4 数据持久化

- [ ] 重启应用后普通会话仍存在
- [ ] 系统通告仍存在且不可误删
- [ ] MCP 配置仍存在
- [ ] 托管任务配置仍存在
- [ ] Skills 启用状态仍存在

---

## 3. 对话能力回归检查

### 3.1 普通对话

- [ ] 发送普通问题可正常得到模型回复
- [ ] 多轮对话上下文保持正常
- [ ] 新建对话、删除对话、切换对话无异常
- [ ] 对话标题自动生成或更新正常

### 3.2 技能识别

- [ ] 问“你有什么 skill，能干什么”可返回真实技能清单
- [ ] 命中某个 Skill 时，不再退化成普通助手自我介绍
- [ ] Skill 说明能影响模型回答内容

### 3.3 即时任务

- [ ] 输入“帮我做一次 ping 测试”等语句可触发即时任务
- [ ] 即时任务结果可正常回到对话中
- [ ] 参数缺失或参数不合法时有明确错误提示

### 3.4 托管任务查询

- [ ] 在对话中可查询托管任务状态
- [ ] 可查询最近异常、最近恢复、最近事件
- [ ] 输出格式具备清晰换行和重点字段展示

---

## 4. Skills 能力回归检查

### 4.1 Skills 扫描与加载

- [ ] 项目内置 Skills 可正常识别
- [ ] `~/.aiops/skills/` 下的用户 Skills 可正常识别
- [ ] Skills 刷新按钮可正常更新结果

### 4.2 Skills 上传与删除

- [ ] 可拖拽上传 Skill 文件夹
- [ ] 可通过选择目录上传 Skill
- [ ] 删除 Skill 后界面和后端状态同步更新

### 4.3 Skills 元信息编辑

- [ ] 说明可正常编辑并保存
- [ ] 标签可正常编辑并保存
- [ ] 修改后刷新仍可保留

---

## 5. 脚本工作台回归检查

### 5.1 页面布局

- [ ] 脚本工作台在常见窗口尺寸下无横向滚动条
- [ ] 即时任务和托管任务列表显示正常
- [ ] 详情区可正常展示选中任务信息

### 5.2 即时任务视图

- [ ] 仅显示脚本名、脚本类型、备注
- [ ] 详情页保留完整信息
- [ ] 标签不在列表页显示，但功能不受影响

### 5.3 托管任务视图

- [ ] 可看到托管任务列表
- [ ] 选中托管任务后可看到状态、日志、配置
- [ ] 异常时卡片和详情页高亮正常

---

## 6. 托管任务回归检查

### 6.1 启动/停止/重启

- [ ] 托管任务可正常启动
- [ ] 托管任务可正常停止
- [ ] 托管任务可正常重启
- [ ] 停止后不会长期卡在“停止中”

### 6.2 配置表单

- [ ] `service_watchdog` 配置可编辑并保存
- [ ] `server_ping` 目标地址可编辑并生效
- [ ] 切换任务后配置不丢失

### 6.3 托管状态

- [ ] 运行中、需关注、告警中、已恢复、已停止状态切换正确
- [ ] 最新日志展示正常
- [ ] 停止后有明确系统日志

### 6.4 守护与恢复

- [ ] 应用重启后，期望运行中的托管任务可自动恢复
- [ ] 应用重启后，运行中的后台托管任务可重新接管
- [ ] 已手动停止的托管任务不会误自动重启
- [ ] 异常退出后，允许自动恢复的任务可按策略重启

### 7.1 MCP Server 管理

- [ ] 自动重连已启用服务正常

- [ ] 托管任务恢复时系统通告收到消息

### 9.1 审计落盘

- [ ] Python 执行会写审计日志
- [ ] 托管任务启动/停止/恢复会写审计日志

- [ ] 工具集成中的审计页可正常打开

- [ ] 概览统计显示正常

- [ ] 输入 taskId / serverName / toolName 可得到相关回放
- [ ] 回放结果能展示事件顺序与核心 payload

## 10. 安全能力回归检查

### 10.1 意图路由与阻断

- [ ] 高风险输入会被阻断
- [ ] 本地优先请求不误下沉到模型
- [ ] 明显提示注入语句能被识别

### 10.2 即时任务安全

- [ ] Skill 参数 schema 校验正常
- [ ] 参数错误不会直接执行脚本
- [ ] 非 instant Skill 不会被即时执行接口误执行

### 10.3 凭据保护

- [ ] API Key 不以明文形式长期保留在 `config.json`
- [ ] 修改模型配置后仍可正常加载和使用

## 11. 构建与交付检查

- [ ] `cargo check` 通过
- [ ] `npm run build` 通过
- [ ] `npm run tauri build` 可生成 `.app`
- [ ] DMG 可正常生成或可通过补充脚本生成
- [ ] 应用图标、窗口标题、产品名称均为 `AIops智能运维中枢`

---

## 12. 建议交付前最小验收结论

建议交付前至少满足以下条件：

1. 普通对话、即时任务、托管任务三条核心链路全部可用
2. MCP 至少完成一条只读工具链路验证
3. 托管任务异常与恢复通知可稳定进入系统通告
4. 审计页能查到至少：
   - 路由事件
   - 托管任务事件
   - MCP 调用事件
5. 重启应用后：
   - 会话不丢
   - 配置不丢
   - 期望运行的托管任务可恢复

满足以上条件后，可视为具备一期交付条件。
