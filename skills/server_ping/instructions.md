# Server Ping Skill

这个 Skill 用来告诉模型如何启动项目中的托管脚本 `scripts/managed/server_ping.py`。

## 适用场景

- 需要持续检测局域网关键地址是否可达
- 需要观察某些固定主机从正常到异常、再到恢复的状态变化
- 需要验证网络连通性托管任务的运行、告警和恢复能力

## 当前默认配置

- 检测目标：
  - `192.168.11.1`
  - `192.168.11.2`
- 检测间隔：`5` 秒
- 连续失败 `3` 次后升级为告警

对应参数来自 `skill.yaml` 里的 `default_args`。

## 使用说明

- 在脚本工作台中找到 `server_ping`
- 点击“启动托管”开始持续检测
- 如果其中任意一个目标无法 ping 通，状态会从运行中变为需关注，再升级为告警
- 当两个目标都重新可达后，状态会变为已恢复

## 给模型的约束

- 这个 Skill 负责描述托管任务的用途和默认参数
- 真正执行的脚本位于项目 `scripts/managed/server_ping.py`
- 如果用户要求持续值守多个固定 IP 的连通性，应优先建议使用这个托管任务
